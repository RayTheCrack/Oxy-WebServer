# WebServer 项目完整重写流程指南

## 一、架构设计

### 系统架构
```
┌─────────────────────────────────────────┐
│           WebServer (主服务器)            │
├─────────────────────────────────────────┤
│  Epoller (I/O 多路复用)                  │
│  HeapTimer (定时器管理)                   │ 网络组件
│  ThreadPool (线程池)                     │
├─────────────────────────────────────────┤
│  HttpConnect (HTTP 连接管理)              │
│  HttpRequest (请求解析)                  │ HTTP组件
│  HttpResponse (响应生成)                 │
├─────────────────────────────────────────┤
│  Buffer (缓冲区管理)                     │
│  SqlConnPool (数据库连接池)              │  基础工具组件
│  Config (配置管理)                       │
└─────────────────────────────────────────┘
```

## 二、开发环境准备

### 环境要求
- **操作系统**：Linux (推荐 Ubuntu 20.04+)
- **编译器**：支持 C++20 的 GCC/G++ (9.0+)
- **数据库**：MySQL 5.7+
- **依赖库**：
  - pthread (线程库)
  - mysqlclient (MySQL 客户端库)

### 项目结构初始化
```bash
# 创建项目目录结构
mkdir -p WebServer/{code/{buffer,config,http,pool,server,timer,log},resources/{css,images,js,video},test,bin,log}

# 创建基础文件
touch WebServer/{CMakeLists.txt,README.md,clear_log.sh}

# 创建WebServer数据库（用于验证用户登入与注册）
mysql -u root -p < init.sql
```

## 三、核心模块开发流程

### 基础工具模块 (优先级：高)

#### 配置管理 (code/config/)
1. **设计要点**：
   - 单例模式，全局唯一配置实例
   - 支持配置文件和命令行参数
   - 类型安全的配置项访问

2. **核心类**：
   - `Config`: 配置管理类，存储所有配置项

3. **实现步骤**：
    main.cpp
    config.h
    config.cpp

#### 缓冲区管理 (code/buffer/)
1. **设计要点**：
   - 动态扩容，避免频繁内存分配
   - 支持从 socket 读写数据
   - 提供高效的字符串查找功能

2. **核心类**：
   - `Buffer`: 缓冲区类，管理读写指针

3. **实现步骤**：
    buffer.h
    buffer.cpp

#### 日志系统 (code/log/)

1. **设计要点**：
   - 异步日志写入，避免阻塞主线程
   - 支持多级别日志 (DEBUG/INFO/WARN/ERROR)
   - 日志文件自动滚动

2. **核心类**：
   - `Logger`: 单例模式，管理日志系统

3. **实现步骤**：
    - blockqueue.h
    - logger.h
    - logger.cpp

### HTTP 协议模块 

#### 数据库连接池 (code/pool/sqlconnpool)
1. **设计要点**：
   - 连接复用，减少连接创建开销
   - 线程安全的连接获取和释放
   - 支持连接数监控

2. **核心类**：
   - `SqlConnPool`: 数据库连接池类
   - `SqlConnRAII`: RAII 风格的连接管理

3. **实现步骤**：
   - 实现连接池初始化
   - 添加连接获取和释放逻辑
   - 实现连接数监控
   - 添加 RAII 管理类

#### HTTP 请求解析 (code/http/httprequest)

1. **设计要点**：
   - 状态机解析 HTTP 请求
   - 支持 GET/POST 方法
   - 支持表单和 JSON 数据解析

2. **核心类**：
   - `HttpRequest`: HTTP 请求解析类

3. **实现步骤**：
   - 实现请求行解析
   - 添加请求头解析
   - 实现请求体解析
   - 添加 URL 解码功能

#### 3.3.2 HTTP 响应生成 (code/http/httpresponse)
1. **设计要点**：
   - 支持多种状态码
   - 自动生成响应头
   - 支持静态文件服务

2. **核心类**：
   - `HttpResponse`: HTTP 响应生成类

3. **实现步骤**：
   - 实现状态行生成
   - 添加响应头生成
   - 实现静态文件服务
   - 添加错误页面处理

#### 3.3.3 HTTP 连接管理 (code/http/httpconnect)
1. **设计要点**：
   - 管理单个 HTTP 连接的生命周期
   - 整合请求解析和响应生成
   - 支持长连接

2. **核心类**：
   - `HttpConnect`: HTTP 连接管理类

3. **实现步骤**：
   - 实现连接初始化
   - 添加请求处理逻辑
   - 实现响应发送
   - 添加连接超时处理
   
###  网络核心模块 

####  线程池 (code/pool/threadpool)
1. **设计要点**：
   - 固定数量的工作线程
   - 任务队列管理
   - 支持任意函数对象提交

2. **核心类**：
   - `ThreadPool`: 线程池类

3. **实现步骤**：
   - 实现基本的工作线程创建
   - 添加任务队列管理
   - 实现任务提交接口
   - 添加线程池停止功能

#### I/O 多路复用 (code/server/epoller)
1. **设计要点**：
   - 封装 epoll 系统调用
   - 提供简洁的接口管理文件描述符

2. **核心类**：
   - `Epoller`: epoll 封装类

3. **实现步骤**：
   - 封装 epoll_create/ctl/wait
   - 实现文件描述符的添加/修改/删除
   - 添加事件处理接口



### 定时器模块 

#### 小顶堆定时器 (code/timer/heaptimer)
1. **设计要点**：
   - 高效的定时器管理
   - 支持超时回调
   - 支持定时器调整

2. **核心类**：
   - `HeapTimer`: 小顶堆定时器类

3. **实现步骤**：
   - 实现小顶堆结构
   - 添加定时器插入/删除/调整
   - 实现超时检查
   - 添加回调函数支持

### 主服务器模块 

#### WebServer 主类 (code/server/webserver)
1. **设计要点**：
   - 整合所有模块
   - 实现事件循环
   - 处理客户端连接

2. **核心类**：
   - `WebServer`: 主服务器类

3. **实现步骤**：
   - 实现服务器初始化
   - 添加事件循环
   - 实现连接处理
   - 添加信号处理
